Translators

Enter translators. They help turning high-level, human-readable block of text we call source code into 0s and 1s for computers to understands. The results of that process are called binaries. To give some very common examples, native desktop applications like Google Chrome, or Notepad are the manifestations of these binaries that we can execute and use without a slight thought of how our computers get all these done

3 types of high-level translators:

  • Interpreters
  • Compilers
  • Hybrid
There are 3 types of high-level computer translators called intepreters, compilers, and hybrid between both interpreters and compilers.

intepreters

For certain languages like Python, we have interpreters which process and run our source code line by line until they encounter an error or the program ends.

Running the whole source code

When using intepreters, passing the whole source code file to the intepreter, which in this case print out 3 different lines of text ...

Passing every single line of code to the interpreter

... is exactly the same as firing up the interpreter and pass in every single line of code for it to translate and instructor the computer what to do. Fun fact, interpreters are also computer programmes, or speaking in what we just learnt from the previous section - binaries, that are the product of compilers translating source codes.

Compliers

Unlike interpreters, compilers gives instructions to our computers in a 2-step process. First, it will translate source code into binary programs. The computer then can understand and execute these binary standalone without any help from the translator. Last but not least, we have our hybrid compilation model, that adds an extra step from when source code is parsed to when instructions are carried out by the computers. This model also introduces an extra actor in the translation process to go from bytecodes to binary instructions.
